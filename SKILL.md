# 导师写作风格蒸馏 Skill

从导师 PDF 论文中系统蒸馏写作风格，生成可运行的「风格卡片」，作为长期写作辅助的认知约束。

**核心理念**：提炼的不是导师说了什么（WHAT），而是导师怎么写（HOW）。

---

## 执行流程

### Phase 0 — 输入确认

收到用户请求后，先确认：

1. **语料来源**：PDF 论文路径？还是需要先收集？
2. **导师信息**：导师姓名、学科领域（影响分析维度权重）
3. **聚焦范围**：全文风格 vs 聚焦某章节（Introduction / Methods 等）？
4. **已有卡片**：是否已有 `advisor_style_card.md`？（有则跳到 Phase 2.5 更新流程）
5. **本地语料提示**：「除了 PDF 论文，导师有没有其他写作素材？比如项目提案、审稿意见、组会 PPT 讲稿等。有的话一起提供，覆盖面更广。」

用户只说了「蒸馏导师风格」没有更多信息 → 默认：PDF 论文 + 全文风格 + 无额外素材，直接推进。

---

### Phase 1 — 解析 & 指标采集

#### Step 1.1：安装依赖并运行脚本

```bash
pip install pymupdf
python3 ~/.workbuddy/skills/advisor-writing-style/scripts/extract_papers.py \
    <PDF目录或单个PDF路径> \
    --output ./parsed_papers
```

脚本输出：
- `<论文名>_text.txt`：按章节结构化文本（含指标）
- `<论文名>_parsed.json`：机器可读结构化数据
- `_summary.json`：跨论文汇总统计
- `_frequent_words.json`：高频词汇 Top 50
- `_connector_words.json`：连接词使用统计

#### Step 1.2：多维度并行分析（核心采集）

读取 `references/style_analysis_framework.md`，沿以下 **6 个采集维度**并行分析：

| Agent | 维度 | 任务 |
|-------|------|------|
| Agent 1 | **宏观结构** | Introduction/Methods/Results/Discussion/Conclusion 的段落数、字数比例、CARS Move 顺序、小节命名风格 |
| Agent 2 | **句式指纹** | 平均句长、长/短句比例、被动语态比例（全局 + 分章节）、时态使用规律、复合句偏好 |
| Agent 3 | **Hedging & 断言** | hedging 词汇频率、assertive 词汇频率、hedging 强度倾向、分章节差异 |
| Agent 4 | **论证逻辑** | 问题引入方式、Contribution 呈现格式、Related Work 组织方式、对比批评语气 |
| Agent 5 | **图表叙事** | Caption 风格、图表引用句式、实验数据描述模板、数值表述格式 |
| Agent 6 | **表达 DNA** | 高频词汇 Top 20、连接词偏好、引用嵌入方式、禁忌词/回避表达、口癖 |

每个 Agent 独立分析后输出结构化报告，主 Agent 汇总。

### Phase 2 — 三重验证 & 风格卡片生成

#### Step 2.1：写作模式三重验证

借鉴 nuwa-skill 的心智模型三重验证，对每个发现的写作模式进行检验：

| 验证 | 标准 | 不通过处理 |
|------|------|-----------|
| **跨篇复现** | 同一模式出现在 ≥50% 的论文中 | 降级为「偶发特征」，不纳入核心风格卡片 |
| **有生成力** | 用此模式能推断导师对新内容的写法 | 保留为「决策启发式」而非核心模式 |
| **有排他性** | 不是所有该领域论文都这样写，体现导师个人偏好 | 标注为「领域通用」还是「导师独特」 |

**通过 0 验证的模式** → 丢弃
**通过 1 验证的模式** → 降级为「偶发特征」，放入附录
**通过 2-3 验证的模式** → 纳入核心风格卡片

#### Step 2.2：矛盾检测

导师不同时期/不同类型论文间可能存在风格矛盾：

| 矛盾类型 | 处理方式 |
|---------|---------|
| **时间演化**（早期 vs 近期风格差异） | 记录演化轨迹，以近 5 年为主，标注「早期特征」 |
| **类型差异**（期刊 vs 会议 vs 书写章节） | 分类型记录，不强行统一 |
| **章节差异**（Introduction 主动多 vs Methods 被动多） | 分章节记录，这是正常现象 |

**绝不做的**：选一边忽略另一边 / 编调和解释 / 假装矛盾不存在

#### Step 2.3：信息不足处理

| 情况 | 处理 |
|------|------|
| 论文 < 5 篇 | 标注「置信度低」，每个模式标注「基于 N 篇样本」 |
| 某章节样本太少（如 Discussion 只在 2 篇中出现） | 该维度降级，不纳入核心卡片 |
| 扫描版 PDF 文本提取失败 | 提示用户先 OCR，标注「此篇未纳入分析」 |

#### Step 2.4：生成风格卡片

读取 `references/skill-template.md`，按标准模板生成 `advisor_style_card.md`。

卡片必须包含：
- **核心风格卡片**：通过三重验证的模式（Phase 2.1）
- **分章节指南**：Introduction / Methods / Results / Discussion / Conclusion 各自的写作偏好
- **表达 DNA 指纹**：句式指纹 + 风格标签 + 口癖 + 禁忌词
- **可复用模板**：Contribution 声明、数据描述、图表引用等句式模板
- **诚实边界**：明确标注哪些维度信息不足、哪些特征置信度低

#### Step 2.5：质量自检

生成后用以下清单自检（来自 `references/style_analysis_framework.md`）：

- [ ] 每个核心模式都通过了 ≥2 重验证？
- [ ] 句式指纹数据来自 ≥5 篇论文？
- [ ] 风格矛盾被识别并分类处理？
- [ ] 信息不足的维度被标注？
- [ ] 删掉导师名字后，风格卡片仍能指导写作（而不只是描述）？
- [ ] 有至少 3 个可直接复用的写作模板？
- [ ] 诚实边界列出了至少 3 条具体局限？

**自检通过 → 交付给用户确认。不通过 → 标注薄弱项，回到 Phase 1 补充采集。迭代上限 2 轮。**

### Phase 3 — 写作辅助（长期使用）

**前提**：已有 `advisor_style_card.md`。若无，先完成 Phase 1-2。

#### 模式 A：润色草稿

用户提供：草稿文本 + 目标章节

1. 读取风格卡片中对应章节的写作偏好
2. 对照风格卡片逐条检查：
   - 句长是否在导师均值 ±20% 范围内
   - 语态是否与该章节习惯一致
   - Hedging 强度是否对齐
   - 连接词、标志性词汇是否使用
   - 是否包含禁忌词
   - Contribution/数据描述是否用了导师的模板句式
3. 输出：润色后版本 + 每处修改的旁注理由 + 偏差报告

#### 模式 B：从零生成

用户提供：内容要点 + 目标章节

1. 将风格卡片作为写作约束
2. 按导师的典型段落结构生成（topic sentence → 证据/展开 → 过渡）
3. 生成后自动执行一轮模式 A 的对齐检查
4. 输出：生成文本 + 自检报告

#### 模式 C：对齐检查

用户提供：已完成的段落/章节

1. 读取风格卡片
2. 输出「偏差报告」：逐条标注偏离项 + 修正建议
3. **不直接修改**，只给建议清单

#### 模式 D：更新风格卡片

触发词：「更新导师风格」「导师发了新论文」「风格卡片需要补充」

1. 对新论文运行 Phase 1 脚本
2. 对比新旧指标，判断：
   - 新信息强化现有模式 → 补充案例
   - 新信息与现有模式矛盾 → 标注变化，更新模型
   - 出现新的写作模式 → 三重验证后决定是否纳入
3. 增量更新 `advisor_style_card.md`，标注更新日期
4. 不重写整个卡片

## 品味守则

| 原则 | 说明 |
|------|------|
| 长文 > 摘要 | 完整的 Introduction/Conclusion 比摘要更能揭示写作习惯 |
| 跨篇 > 单篇 | 同一模式在多篇中出现才是真风格 |
| 演化 > 静态 | 风格变化的地方比一直不变的部分更有信息量 |

### 绝不做

- 编造导师没有的写作偏好
- 把领域通用写法包装成导师的「独特风格」
- 在论文样本不足时强行输出高置信度结论
- 忽略风格矛盾，假装一致性

## 特殊场景

### 扫描版 PDF
脚本提取为空时 → 提示用户先 OCR，推荐 `marker` 或 `Mathpix`
标注此篇「未纳入分析」，不强行生成

### 多导师合著论文
如果论文是导师与学生合著，写作风格可能混杂：
- 优先选择**导师为唯一通讯作者**的论文
- 如果只有合著论文，标注「合著论文，风格可能混杂」，降低置信度

### 非英文论文
如果导师主要发中文论文：
- 分析框架中的时态/被动语态维度需要调整
- 连接词/hedging 词汇替换为中文对应项
- 风格卡片模板按中文论文格式调整
